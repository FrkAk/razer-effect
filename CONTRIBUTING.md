# Contributing

## Workflow

1. **Open an issue first** — describe the bug or feature before writing code. This prevents wasted effort if the direction doesn't fit the project.
2. **Fork the repo** — you won't have direct write access; all changes come through forks.
3. **Create a branch** on your fork (e.g. `fix/fade-timer-overflow`).
4. **Make your changes** — keep them focused on a single concern.
5. **Open a PR** from your fork's branch targeting `main` here.
6. **Wait for review** — all PRs require approval before merge. Squash merge is applied.

## Requirements

- A supported Razer device with OpenRazer installed and daemon running
- Python 3.10+, Fedora/Linux recommended

## Code style

- Match the existing style — no new abstractions for single-use code
- No new dependencies without prior discussion in the issue

## Scope

This project is intentionally minimal. Large feature expansions (new device ecosystems, new UIs) are likely out of scope — confirm in an issue before building.
